[1m  exit(Dest, Reason)[0m

  Old form of [4mexit_signal/2[0m, with a quirk when sender and receiver
  are the same.

  [4mNote[0m

    The function [4merlang:exit/2[0m is named similarly to [4m[0m
    [4merlang:exit/1[0m but provides very different functionality. The [4m[0m
    [4merlang:exit/1[0m function should be used when the intent is to
    stop the current process by raising an exception of class [4m[0m
    [4mexit[0m.

    The [4merlang:exit_signal/2[0m function, or the old form [4m[0m
    [4merlang:exit/2[0m, should be used when the intent is to send an
    exit signal to another process. Note also that [4merlang:exit/1[0m
    raises an exception that can be caught, while [4m[0m
    [4merlang:exit_signal/2[0m does not cause any exception to be
    raised.

  [4mWarning[0m

    This function has a quirk: When a process [4mP[0m sends an exit
    signal with reason [4mnormal[0m to itself using this function,
    that is, [4merlang:exit(self(), normal)[0m, the behavior is as
    follows:

     • [4mP[0m exits with reason [4mnormal[0m if [4mP[0m is not trapping
       exits.

     • If [4mP[0m is trapping exits, the exit signal is transformed
       into a message [4m{'EXIT', From, normal}[0m, where [4mFrom[0m is [4m[0m
       [4mP[0m's process identifier, and delivered to [4mP[0m's message
       queue. Note that this differs from when a process sends
       an exit signal with reason [4mnormal[0m to another process
       than itself (see [4mexit_signal/2[0m for details). This
       behavior is kept for backward compatibility reasons. Use [4m[0m
       [4mexit_signal/2[0m for new code.
