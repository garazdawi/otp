[1m  binary_to_atom(Binary)[0m

[4mSince[0m:
  OTP 23.0

  There is no documentation for binary_to_atom(Binary, utf8)

[1m  binary_to_atom(Binary, Encoding)[0m

  Returns the atom whose text representation is [4mBinary[0m, creating a
  new atom if necessary.

  If [4mEncoding[0m is [4mutf8[0m or [4municode[0m, the binary must contain
  valid UTF-8 sequences.

  [4mNote[0m

    Note that once an atom is created, it cannot be deleted. The
    Erlang system has a configurable limit on the number of atoms
    that can exist. To avoid reaching this limit, consider whether [4m[0m
    [4mbinary_to_existing_atom/2[0m is a better choice than [4m[0m
    [4mbinary_to_atom/2[0m. The number of characters that are permitted
    in an atom name is limited.

  [4mChange[0m

    As from Erlang/OTP 20, [4mbinary_to_atom(Binary, utf8)[0m is
    capable of decoding any Unicode character. Earlier versions
    would fail if the binary contained Unicode characters > 255.

[1mExamples[0m

    1> binary_to_atom(<<"Erlang">>, latin1).
    'Erlang'
    2> binary_to_atom(<<960/utf8>>, utf8).
    'π'
